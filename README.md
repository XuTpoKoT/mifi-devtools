# Сервис OTP-кодов

## Функциональность
Реализовано API пользователя, которое позволяет:

- Генерировть OTP-код, привязанный к идентификатору операции.
- Валидировать OTP-код, который был выслан пользователю.

Реализовано API администратора, которое позволяет:

- Менять конфигурацию OTP-кодов (время жизни и количество знаков в коде).
- Получать список всех пользователей кроме администраторов.
- Удалять пользователей и привязанные к ним OTP-коды.

Также реализована регистрация и авторизация пользователей по логину и паролю.
После авторизации пользователь получает JWT-токен, который должен использоваться для дальнейших запросов.

## Стек технологий
В качестве системы сборки использовался gradle.
Для разработки API был использован Spring MVC.
Было реализовано подробное покрытие логами каждого запроса к API с помощью SLF4.

## Структура приложения
- Слой API, содержащий обработчики HTTP-запросов.
- Слой сервисов, содержащий в себе основную бизнес-логику приложения.
- Слой DAO, содержащий в себе классы, осуществляющие выполнение запросов к БД.

## Запуск приложения

Выполните команду в корне проекта:
   ```bash
   docker-compose up -d
   ```
Сервер запустится на `http://localhost:8080`.

База данных PostgreSQL инициализируется автоматически.

Swagger будет доступен  на `http://localhost:8080/swagger.html`

API приложения описано в openapi.yaml
